# content-queue — 예약 발행 대기열

여기에 미리 쓴 글(`.md`/`.mdx`)을 넣어두면, **매일 09:00 KST에 GitHub Actions가 1편씩** `src/content/posts/`로 옮겨 발행하고 push합니다(→ Cloudflare 자동 배포).

이 폴더는 Astro 콘텐츠 컬렉션 **밖**이라, 대기 중인 글은 절대 사이트에 빌드/노출되지 않습니다.

## 사용법

1. 이 폴더에 글 파일을 추가합니다. 프론트매터는 일반 글과 동일:

   ```
   ---
   title: '제목'
   description: '설명'
   tags: ['frontend']
   ---
   본문…
   ```
   - `pubDate`는 **써도 되고 안 써도 됩니다** — 발행 시 자동으로 그날 날짜(KST)로 덮어써집니다.
   - `draft`, `heroImage`, `series` 등 나머지 프론트매터는 그대로 유지됩니다.

2. **발행 순서 = 파일명 오름차순.** 순서를 통제하려면 숫자 접두어를 씁니다:
   - `001-first-post.md`, `002-second-post.md` …
   - 선행 숫자 접두어(`001-`)는 발행 시 **URL slug에서 자동 제거**됩니다 → `/posts/first-post/`

3. 커밋해서 push하면 대기열에 등록 완료. 그날그날 하나씩 빠져나갑니다.

## 참고

- **큐가 비면** 워크플로는 아무 커밋도 하지 않고 조용히 넘어갑니다(에러 아님).
- **즉시/테스트 발행**: GitHub → Actions → "Daily post" → _Run workflow_ (수동 실행).
- 프론트매터가 잘못되면 그날 사이트 빌드가 실패할 수 있습니다(라이브는 직전 배포 유지). 넣기 전에 형식을 확인하세요.
- 크론은 GitHub 부하에 따라 몇 분 지연될 수 있습니다.
